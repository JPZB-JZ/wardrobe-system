# 编程规则

## 代码规范
1. 使用 TypeScript 强类型，禁止 `any`（除非与第三方交互）
2. React 组件使用函数式 + Hooks，禁止 class 组件
3. 状态管理统一使用 Zustand，禁止 prop drilling 超过 2 层
4. CSS 使用 TailwindCSS utility-first，复杂样式抽成 `@layer components`
5. 文件命名：组件 PascalCase，工具/服务 camelCase

## 目录规则
- `components/` — 可复用的 UI 组件（Button, Modal, Toast 等）
- `pages/` — 路由页面组件，每个页面一个文件
- `store/` — Zustand store，按功能拆分
- `services/` — API 调用、AI 服务、第三方集成
- `hooks/` — 自定义 React Hooks
- `utils/` — 纯函数工具（不依赖 React）

## Git 规范
- feat: 新功能
- fix: 修复
- style: 样式调整
- refactor: 重构
- docs: 文档

## APK 注意事项
- 所有路径使用相对路径（vite.config 的 `base: './'`）
- 使用 HashRouter（不是 BrowserRouter），避免 file:// 路由问题
- 图片存储为 base64 dataURL（避免文件系统权限）
- fetch API 调用需要在 capacitor.config.ts 中配置允许的域名
- 不依赖 cookie/session，所有状态在 localStorage
